Course Description

CSCI-C 343 Data Structures (4 cr.) P: CSCI-C 212. CSCI-C 241. C: CSCI-C 241. N & M Systematic study of data structures encountered in computing problems, structure and use of storage media, methods of representing structured data, and techniques for operating on data structures. Lecture and laboratory. Credit given for only one of CSCI-C 343 or H 343.

At the end of this course you will have acquired a firm knowledge of data structures and practical experience using their implementation in the Java Collections Heirarchy. You will learn industry-leading tools and collaborative practices. Additionally, the core Computer Science curriculum will be complete.
Pretty awesome.

Topics

This course will cover data structures and their algorithms. We will discuss their implementation in the Java Collections Heirarchy and analysis of their runtime performance and memory costs.

Data Structures
  • Arrays
  • Lists (growable arrays)
  • Stacks & Queues
  • Binary Search Trees
  • AVL Trees
  • Heaps
  • Binomial Forests
Analysis
  • Big-O Notation
  • Memory Cost
  • Runtime Cost
Miscellaneous
  • Distributed Version Control
  • Java Debugging

Assignments

Analysis Skills

Assignments will be made to cover the analysis of runtime and memory efficiency for each data structure. These are to be completed individually (that is, not as a group), and will be turned in during Wednesday lecture, when assigned. An analysis assignment and a programming assignment will never be due in the same week, although multiple assignments may be released at the same time.

Project

To understand the data structures taught in this course, students will implement a library-based media player in Java using the gstreamer library. This will illustrate the principles taught in lecture. This project will foster group collaboration skills, vital to success as a computer scientist working in industry and academia. Teams will be composed of 2-3 students enrolled in the same lab section, such that lab time can be used to help foster collaboration and give instructors an opportunity to observe group participation. The project is cumulative, and the final assignment will involve a creative extension of the media player, to be determined by each group and will be due during the week of December 3-7, graded during a live demo to course instructors.

GitHub:IU

Software development will use the Git Source Control Manager, used by Facebook, Twitter, Microsoft, LinkedIn, Mozilla, the entire Linux kernel and many, many more. We will use a private instance of the GitHub software which uses your IU credentials, known as GitHub:IU. See the IU Knowledge Base article What is Git and GitHub:FI, and how do I use them? This is not optional, as assignments will be turned in via a git commit.

Tools

It is recommended that students use an IDE with an appropriate Git plugin to help manage version control. It is recommended that development occur in the Mine, located in Lindley Hall 035. It is the student's own perogative to install the development environment on their personal machine. Recommendations will be given in the first week of class, along with configuration instructions.

Deadlines

Assignments will be turned in via a commit to GitHub:IU by 11:59pm on Wednesday evenings. This process will be explained during the first lab section. Important information for the next assignment will be given during lab sections on Thursday and Friday, to encourage an early start. New assignments will be released on Mondays.

Tests

The course will have two tests — a midterm and a final — primarily over the analysis portions of the course and the Java Collections Heirarchy, along with some understanding of the gstreamer API from the project.

The final will be Friday, December 14 from 5-7pm. It is in the syllabus 4 months in advance. Do not leave Bloomington before your final.
