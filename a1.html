  2. reccommended - Passwordless Login

You will need to register different keys or copy over your .ssh folder from the Burrow. Be warned that you might overwrite your existing keys, so check if you already have a pair made.

Setup repoitory

Pick 1 member to create a GitHub:IU repository called "c343player". Instead of the instructions under "Next steps:", the team host should do the following: 

mkdir c343player
cd c343player
git init
touch README.md

Then, edit the README.md using whichever editor you want, using the following format. 

# C343 Player: [GROUP_NAME]
[NAME] (`[IU ID]`)

## Summary
[PROJECT SUMMARY]

## Goals
[PROJECT GOALS]
For advanced formatting, use GitHub Flavored Markdown in your file.

After this, commit the file and push to GitHub:IU: 

git add README.md       
git commit -m 'first commit'
git remote add origin git@github.iu.edu:YOUR IU USERNAME/c343player.git
git push origin master

Other team members will need to log into GitHub:IU and then the team lead will add the members to the repository through the Admin page's Collaborators tab on the c343player repo.

The other team members will clone this repository: 

git clone git@github.iu.edu:LEADER'S IU USERNAME/c343player.git
Open the README.md and after the previous person's name and IU ID, add your name and ID: [NAME] (`[IU ID]`). Then add the file to the staging area, commit, and push: 
git add README.md       
git commit -m 'adding name'
git push

Example of a completed README.md.
